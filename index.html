    REST INTERFACE

    • ./site/site/siteame//bookname/{$Name} - causes a generalised site mechanism. This displates all information in site/bookname open at page $Name useing a templte
    • Name.ehe?arg1=... causes ehe to be expanded
    • Name.md?arg1 ... causes ehe expansion followd by markdown expansion
    • site/mod/modname/func?arg1=1,&arg2 causes erlang module to be called

    Design

    1. The webserver is a single file called webserver
    2. eserver -conf file.conf with no config standard values are assumed
    3. The root is CWD
    4. Must be a directory called sites - or an entire web site In a zip file (called mysite.zip)

    Logic

    If we use the site interface we do the following:

    1. Look in site/template/page.ehe this contins a template with variables Content, Sidbar
    2. Set SideBar = the *'s in /site/books/ideas/*
    3. Let X = /site/books/ideas/one.md
    4. Let X1 = ehe_exand(X, SYS ++ Args)
    5. Let Content = md2html(X1)
    6. Let Page = ehe_expand(template, #{Content => ...}
